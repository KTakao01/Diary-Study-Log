## 改善(AI)

swap簡単にする、くらい。
自分自身と比べる必要ないのでi+1スタート

```python
def selection_sort(a, n):
    for i in range(0, n-1):
        min_index = i
        for j in range(i+1, n):
            if a[j] < a[min_index]:
                min_index = j

        a[i], a[min_index] = a[min_index], a[i]

        print(*a)

n = int(input())
a = list(map(int, input().split()))

selection_sort(a, n)
```

## 疑問1(AI回答)

### 答え：n-2まで終わった時点でn-1は自動的に確定してるから

`n=5` の例で考えると：

```
i=0: A[0]~A[4]の最小値をA[0]に置く → A[0]確定
i=1: A[1]~A[4]の最小値をA[1]に置く → A[1]確定
i=2: A[2]~A[4]の最小値をA[2]に置く → A[2]確定
i=3: A[3]~A[4]の最小値をA[3]に置く → A[3]確定
                                          ↓
                              残りA[4]は1個しかないので自動的に最大値 = 確定
```

A[0]～A[3]が確定した時点で、残り1個しかないA[4]は**「残った中で一番大きい値」に決まってる**ので、わざわざチェックする必要がないんです。

---

### 一言まとめ

`n-1` 個が確定すれば最後の1個は**勝手に確定する**ので、ループは `n-2` まで で十分ということです。

### 役割が違うから範囲が違う

```
外側ループ (i): 「どこに最小値を置くか」の位置を決める
内側ループ (j): 「最小値がどこにあるか」を探す
```

---

### 外側が n-2 までの理由

さっきの話と同じで「最後の1個は自動確定」だから。

```
i=0: 0番目に置く最小値を探す
i=1: 1番目に置く最小値を探す
...
i=n-2: n-2番目に置く最小値を探す → 残りn-1番目は自動確定
```

---

### 内側が n-1 までの理由

内側は**配列の端っこまで全部チェック**しないと最小値を見つけられないから。

```
i=0のとき、j は 1 〜 n-1 まで全部見る必要がある

[ 3, 1, 4, 1, 5 ]
  ↑  ←←←←←←← j がここまで全部チェック
  i=0             ↑
                 n-1
```

---

### 一言まとめ

| ループ | 役割                 | 範囲                     |
| ------ | -------------------- | ------------------------ |
| 外側 i | **置く場所**を決める | 0 〜 n-2（最後は不要）   |
| 内側 j | **最小値を探す**範囲 | i+1 〜 n-1（端まで全部） |

「置く場所は手前まででいい、でも探す範囲は端まで全部必要」という違いです！

## 疑問2(AI回答)

### n-i-1 回の意味

内側ループは `j = i+1 から n-1 まで` 回ります。

```
j の個数 = (n-1) - (i+1) + 1 = n-i-1 回
```

### 具体例で確認（n=5）

```
i=0: j が 1~4 → 4回   (= 5-0-1)
i=1: j が 2~4 → 3回   (= 5-1-1)
i=2: j が 3~4 → 2回   (= 5-2-1)
i=3: j が 4~4 → 1回   (= 5-3-1)

合計: 4+3+2+1 = 10回
```

### 直感的に言うと

iが増えるにつれて**「整列済み範囲が広がる」＝「未整列範囲が狭まる」**ので、内側ループの回数がどんどん減っていく、というだけです。
